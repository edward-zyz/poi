# 餐饮门店选址辅助工具 (Location Scout) 🚀

欢迎来到 **Location Scout**！这是一个基于高德地图 API 的餐饮选址辅助工具，旨在通过 POI（兴趣点）大数据分析，帮助餐饮从业者科学评估开店位置。

无论你是想学习全栈开发，还是想直接将其投入实战，本指南都将帮助你快速在本地搭建并运行起这套系统。

---

## 🛠 快速上手指南

### 1. 环境准备
在开始之前，请确保你的电脑已安装以下基础环境：
- **Node.js**: 推荐版本 v18.x 或更高 (可以通过 `node -v` 查看)。
- **npm**: 随 Node.js 一起安装。
- **高德地图 API Key**: 你需要前往 [高德开放平台](https://lbs.amap.com/) 注册并申请一个 **Web 服务** 类型的 Key。

### 2. 获取代码与安装依赖
```bash
# 克隆/下载项目后进入目录
cd POI

# 安装全量依赖 (根目录下运行即可，会自动处理前端和后端)
npm install
```

### 3. 核心配置 (关键)
项目需要配置高德地图密钥才能正常工作。

**后端配置：**
1. 找到 `config/gaode.config.json.example` (如果没有，直接创建 `config/gaode.config.json`)。
2. 填写你的 API Key：
   ```json
   {
     "apiKey": "你的高德REST_API_KEY",
     "securityJsCode": "你的安全密钥_JS_CODE"
   }
   ```

**前端配置：**
1. 进入 `frontend` 目录。
2. 创建 `.env.local` 文件并添加：
   ```env
   VITE_GAODE_KEY=你的前端JS_API_KEY
   ```

### 4. 启动服务
建议打开两个终端窗口，分别运行：

- **终端 1 (后端):**
  ```bash
  npm run dev:backend
  ```
- **终端 2 (前端):**
  ```bash
  npm run dev:frontend
  ```

启动成功后，打开浏览器访问 `http://localhost:5173` 即可看到精美的地图交互界面！

## 📖 核心功能使用说明

为了让你（或你的朋友）能快速上手，这里列出了 Location Scout 的三大核心使用场景：

### 场景 1：商圈热力大盘（全局洞察）
*   **操作**：在顶部搜索栏输入你想调研的**主品牌**（如：`喜茶`）和**竞品关键词**（如：`奈雪的茶, 霸王茶姬`），点击“生成分析”。
*   **效果**：地图上会立即呈现出品牌的分布热力。
    *   **颜色深浅**代表了门店密集的程度。
    *   通过左侧图层控制，你可以随时开关主品牌或竞品的视觉显示，直观对比“势力范围”。

### 场景 2：目标点位深度穿透（单点评估）
*   **操作**：如果你看中了一个具体的铺位，直接在地图上**点击该位置**。
*   **效果**：
    *   **分析圈**：系统会自动以点击点为中心，画出一个半径 1000 米的分析圈。
    *   **数据面板**：右侧会弹出详尽的报表，展示该点位周边的门店总数、平均密度，以及核心竞品的具体名称。
    *   **智能评分**：根据周边竞争情况自动评估密度等级（高/中/低），辅助你判断这里是“蓝海”还是“红海”。

### 场景 3：选址库管理（决策沉淀）
*   **操作**：在目标点位分析面板中，点击“加入规划”。
*   **效果**：
    *   该点位将被永久收藏到左下角的“点位规划”列表中。
    *   你可以为不同的候选铺位添加备注信息（如：`租金 2W/月`、`带上下水`）。
    *   支持设置 **优先级**（P0-P2），把最有潜力的位置排在前面，方便团队沟通决策。

## 🏗 技术架构

- **前端**: React + Vite + Zustand + AMap JS SDK (现代、极速、响应式)
- **后端**: Node.js + Express + TypeScript (结构清晰、类型安全)
- **数据库**: SQLite (轻量级，无需安装数据库软件，即插即用)
- **缓存**: 基于本地 SQLite 的 POI 数据缓存，减少 API 调用频次，降低成本。
